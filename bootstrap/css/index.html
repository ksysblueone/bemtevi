CSS

Configurações básicas do CSS, estilos fundamentais para elementos HTML com classes auxiliares, e sistema avaçado em grade.

Sistema em grade

O Bootstrap inclui um sistema em grade que comporta até 12 colunas de acordo com o tamanho da tela. Com isso existem classes pré definidas que facilitam o layout.

Introdução

  • Rows (.row) devem ser usadas dentro de um container (.container-fluid) para alinhamento e espaçamentos corretos.
  • Utilize rows para criar grupos horizontais de colunas.
  • Exemplificando, rows podem ser linhas de uma planilha onde as colunas (.col-*) são suas células.
  • O conteúdo da página deverá ficar dentro das colunas e somente colunas podem vir imediatamente como filhas após a criação de uma row.
  • Classes pré definidas como .row e .col-xs-4 estão disponíveis para rapidatemente criar um layout em grade.
  • Colunas criam automaticamente espaços entre as demais colunas utilizando-se de padding.
  • As colunas da grade são criadas especificamente com um número de 12 disponíveis. Por exemplo, três colunas .col-xs-4 terão o mesmo tamanho e totalizarão o tamanho máximo de 12 colunas.
  • Se existir mais de 12 colunas, as mesmas serão colocadas em uma nova linha.

Media queries

O bootstrap utiliza de media queries para determinar os tamanhos de acordo com a resolução da tela.

/* Como já explicado acima, no conceito mobile first todas as classes fora de media queries são aplicadas para pequenos dispositivos (celulares, menores que 768px) */
/* Sem media queries é o padrão do Bootstrap */

/* Para pequenos dispositivos, maiores que celulares como tablets, é usada a media querie a seguir (tablets, com 768px e maiores): */
@media (min-width: 768px) { ... }

/* Para dispositivos médios (notebooks, com 992px e maiores): */
@media (min-width: 992px) { ... }

/* Para dispositivos grandes (monitores, tvs, 1200px e maiores): */
@media (min-width: 1200px) { ... }

Ocasionalmente pode existir limites, incluindo a condição max-width para limitar o CSS para um determinado tamanho de dispositivo.

@media (max-width: 768px) { ... }
@media (min-width: 768px) and (max-width: 992px) { ... }
@media (min-width: 992px) and (max-width: 1200px) { ... }
@media (min-width: 1200px) { ... }

Exemplo: Envolvendo as colunas

Se mais de 12 colunas estiverem dentro de uma única linha (.row), cada grupo de coluna irá como uma unidade "quebrar" em uma nova linha.

.col-xs-9
.col-xs-4
Sendo 9 + 4 = 13 > 12, essas 4 colunas iniciarão uma nova linha, mas continuarão dentro da .row como uma unidade.
.col-xs-6
As colunas subsequentes continuarão ao longo desta linha.

Para mudar a ordem das colunas para uma determinada resolução, basta utilizar as classes com .col-md-push-* e/ou .col-md-pull-* para alterar.

.col-md-3 .col-md-pull-9

Formulários

Exemplo básico

Alguns elementos de formulário recebem um estilo global automaticamente. Todos os <input>, <textarea> e <select> com .form-control recebem width: 100%; por padrão. Envolva os labels e controles com .form-group para otimização do espaço.

Exemplo de bloco com texto auxiliar.

<form>
  <div class="form-group">
    <label for="exampleInputEmail1">E-mail</label>
    <input type="email" class="form-control" id="exampleInputEmail1" placeholder="E-mail">
  </div>
  <div class="form-group">
    <label for="exampleInputPassword1">Senha</label>
    <input type="password" class="form-control" id="exampleInputPassword1" placeholder="Senha">
  </div>
  <div class="form-group">
    <label for="exampleInputFile">Anexo</label>
    <input type="file" id="exampleInputFile">
    <p class="help-block">Exemplo de bloco com texto auxiliar.</p>
  </div>
  <div class="checkbox">
    <label>
      <input type="checkbox"> Marque-me
    </label>
  </div>
  <button type="submit" class="btn btn-default">Enviar</button>
</form>

Não misture grupos de formulários com grupos de input

Não misture grupos de formulários diretamente com grupos de input. Ao invés disso, inclua um grupo de input dentro de um grupo de formulário.

Formulário em linha

Adicione .form-inline ao formulário (não precisa ser necessariamente um <form>) para um formulário alinhado a esquerda e com blocos em linha. Este estilo é apenas aplicado quando a resolução da tela for maior que 768px.

Pode exigir larguras personalizadas

Inputs e selects tem width: 100%; aplicado por padrão no Bootstrap. Com o formulário em linha, é resetado para width: auto; assim outros controles podem residir na mesma linha. Dependendo do layout, adicionar largura personalizada pode ser requerida.

<form class="form-inline">
  <div class="form-group">
    <label for="exampleInputName2">Nome</label>
    <input type="text" class="form-control" id="exampleInputName2" placeholder="John Doe">
  </div>
  <div class="form-group">
    <label for="exampleInputEmail2">E-mail</label>
    <input type="email" class="form-control" id="exampleInputEmail2" placeholder="john.doe@exemplo.com.br">
  </div>
  <button type="submit" class="btn btn-default">Enviar convite</button>
</form>
<form class="form-inline">
  <div class="form-group">
    <label class="sr-only" for="exampleInputEmail3">E-mail</label>
    <input type="email" class="form-control" id="exampleInputEmail3" placeholder="E-mail">
  </div>
  <div class="form-group">
    <label class="sr-only" for="exampleInputPassword3">Senha</label>

Controles suportados

Exemplos de controles de formulário padrão suportados pelo Bootstrap.

Inputs

O mais comum dos controles de formulário, input do tipo text. Incluindo suporte para todos os elementos HTML5: text, password, datetime, datetime-local, date, month, time, week, number, email, url, search, tel, e color.

Declaração do tipo é obrigatória

Inputs apenas vão ser estilizados totalmente se o type estiver corretamente declarado.

<input type="text" class="form-control" placeholder="Input do tipo text">

Grupos de input

Para adicionar texto ou botões antes e/ou depois de qualquer input de texto <input>, veja o componente grupos de input.

Textarea

Controle de formulário com suporte para múltiplas linhas de texto. Altere o atributo rows conforme for necessário.

<textarea class="form-control" rows="3"></textarea>

Checkboxes e radios

Checkboxes são opções para selecionar um ou mais opções de uma lista, enquanto radios são para selecionar apenas uma opção entre muitas.

Um checkbox ou radio com o atributo disabled será estilizado apropriadamente. Para ter o <label> para o checkbox ou radio com o cursor de "não permitido" quando o usuário mover o mouse em cima, adicione a classe .disabled no seu .radio, .radio-inline, .checkbox, .checkbox-inline, ou <fieldset>.

Padrão

</div>
<div class="checkbox disabled">
  <label>
    <input type="checkbox" value="" disabled>
    Opção dois está desabilitada
  </label>
</div>

<div class="radio">
  <label>
    <input type="radio" name="optionsRadios" id="optionsRadios1" value="option1" checked>
    Opção um é isso e aquilo &mdash; não deixe de incluir porque é ótimo.
  </label>
</div>
<div class="radio">
  <label>
    <input type="radio" name="optionsRadios" id="optionsRadios2" value="option2">
    Opção dois pode ser outra coisa e selecionando-o irá desmarcar a opção um.
  </label>
</div>
<div class="radio disabled">

    <input type="radio" name="optionsRadios" id="optionsRadios3" value="option3" disabled>
    Opção três está desabilitada
  </label>
</div>

Botões

Atributos dos botões

Para botões utilize os elementos <a>, <button>, ou <input>.

Link 
<a class="btn btn-default" href="#" >Link</a>
<button class="btn btn-default" type="submit">Botão</button>
<input class="btn btn-default" type="button" value="Input">
<input class="btn btn-default" type="submit" value="Enviar">

Opções

Utilize qualquer uma das classes de botões dispoíveis para adicionar o estilo rapidamente.

<!-- Botão padrão -->
<button type="button" class="btn btn-default">Padrão</button>

<!-- Fornece um visual extra e identifica uma ação primária em um conjunto de botões -->
<button type="button" class="btn btn-primary">Primário</button>

<!-- Indica um botão de sucesso ou uma ação positiva -->
<button type="button" class="btn btn-success">Sucesso</button>

<!-- Botão para mensagens de informações ou alerta informativo -->
<button type="button" class="btn btn-info">Info</button>

<!-- Indica uma ação que necessita de certa atenção -->
<button type="button" class="btn btn-warning">Atenção</button>

<!-- Indica uma ação cautelosa ou potencialmente negativa -->
<button type="button" class="btn btn-danger">Perigo</button>

<!-- Enfatiza um botão com uma aparência de link, enquanto mantém característica de botão -->
<button type="button" class="btn btn-link">Link</button>

Imagens responsivas

Imagens no Bootstrap podem ser responsivas adicionando a classe .img-responsive. Isso aplica max-width: 100%;, height: auto; e display: block; para a imagem fazendo com que ela se ajuste corretamente ao elemento pai.

Para centralizar as imagens utilizando a classe .img-responsive, utilize .center-block ao invés de .text-center. Veja a sessão de classes auxiliares para mais detalhes sobre o uso do .center-block.

<img src="..." class="img-responsive" alt="Imagem responsiva">

A partir da versão 3.2.0 do Bootstrap, as classes .visible-*-* vêm com três variações, uma para cada tipo de display listadas abaixo.

Grupo de classes CSS display
.visible-*-block display: block;
.visible-*-inline display: inline;
.visible-*-inline-block display: inline-block;

Para disposivitos extra pequenos (xs) por exemplo, também estão disponíveis classes .visible-*-* que são: .visible-xs-block, .visible-xs-inline, e .visible-xs-inline-block.

As classes .visible-xs, .visible-sm, .visible-md, e .visible-lg também existem, mas foram descontinuadas a partir da versão 3.2.0.
